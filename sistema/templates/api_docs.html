Visão Geral

A API REST do Sistema de DRE permite integrar dados financeiros com outros sistemas. Todas as requisições devem incluir autenticação via chave de API. O fuso horário padrão é America/Sao_Paulo (UTC-3). Intervalo de anos suportado: {{ ANO_INICIO }}–{{ ANO_FIM }}.

Autenticação

Inclua a chave de API em todas as requisições de uma das formas:

Header: X-API-Key: snm_sua_chave_aqui Query string: ?api_key=snm_sua_chave_aqui
URL Base
{{ request.url_root }}api/v1/
Respostas

Todas as respostas são em formato JSON (UTF-8). Em caso de erro, o campo erro conterá a descrição.

Endpoints de Leitura
GET /api/v1/status Sem autenticação
Verifica se o sistema está operacional.
{ "status": "ok", "versao": "3.0", "anos": "{{ ANO_INICIO }}-{{ ANO_FIM }}", "fuso": "America/Sao_Paulo" }
GET /api/v1/dre/{loja_id}/{ano}/{mes} read
Retorna o DRE completo de uma loja em determinado mês/ano.
ParâmetroTipoDescrição
loja_idintID da loja
anointAno ({{ ANO_INICIO }}–{{ ANO_FIM }})
mesintMês (1–12)
{ "loja_id": 1, "ano": 2026, "mes": 3, "nome_mes": "Março", "faturamento_bruto": 45000.00, "total_taxas": 2100.50, "faturamento_liquido": 42899.50, "total_despesas": 18500.00, "royalties": 1500.00, "resultado": 22899.50, "margem_percentual": 50.89, "dias_com_lancamentos": 28 }
GET /api/v1/resumo-anual/{loja_id}/{ano} read
Retorna o resumo de todos os 12 meses do ano para a loja.
GET /api/v1/lojas read
Lista todas as lojas ativas com ID, nome, CNPJ e razão social.
GET /api/v1/formas-pagamento read
Lista formas de pagamento ativas com taxa.
GET /api/v1/categorias-despesa read
Lista categorias de despesa ativas com tipo.
GET /api/v1/plataformas read
Lista plataformas/apps ativos.
